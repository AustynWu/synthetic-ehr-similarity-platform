Synthetic vs Real Health Record Similarity Evaluation Platform

1. Metric selection should not be a flat list.

TASK 1 — Metric definitions
==================================================

Find the current metric list or create a typed metric definition file if needed.

Each metric should have:
- key
- label
- description
- group
- subgroup
- applicableVariableTypes
- priority

Suggested types:

type MetricGroup = "Univariate" | "Multivariate";

type MetricSubgroup =
  | "Numerical"
  | "Categorical"
  | "Numerical–Numerical"
  | "Categorical–Categorical"
  | "Mixed";

type VariableType =
  | "continuous_numerical"
  | "discrete_numerical"
  | "categorical"
  | "unknown";

interface MetricDefinition {
  key: string;
  label: string;
  description: string;
  group: MetricGroup;
  subgroup: MetricSubgroup;
  applicableVariableTypes: VariableType[];
  priority: "Core" | "Recommended" | "Optional";
}

Metrics to include:

Univariate > Numerical:
- Mean Difference
- Median Difference
- Standard Deviation Difference
- IQR Difference
- KS Statistic
- Wasserstein Distance

Univariate > Categorical:
- Category Proportion Comparison
- Chi-square Statistic
- Unseen / Invalid Category Check
- Total Variation Distance, if already supported

Multivariate > Numerical–Numerical:
- Pearson Correlation Comparison
- Correlation Matrix Distance
- Spearman Correlation Comparison, if already supported

Multivariate > Categorical–Categorical:
- Cramér’s V Comparison
- Selected Joint Distribution Comparison

Multivariate > Mixed:
- Group-wise Summary Comparison
- Correlation Ratio Eta
- Mutual Information, if already supported

If a metric is not implemented in backend yet, it can still exist in mock/frontend as planned MVP UI, but do not claim backend calculation is complete.
